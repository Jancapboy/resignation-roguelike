# 程序员离职模拟器 Roguelike

## PRD - 产品需求文档

### 1. 游戏概述

**游戏名称**: 离职模拟器 (Resignation Roguelike)  
**类型**: Roguelike / 生存模拟 / 黑色幽默  
**平台**: Web (优先) / 桌面  
**核心体验**: 在高压 IT 环境中生存，管理多重资源，决定何时离职

### 2. 核心机制

#### 2.1 四维资源系统

| 资源 | 英文 | 范围 | 说明 |
|------|------|------|------|
| 理智值 | sanity | 0-100 | 精神健康，归零=崩溃离职 |
| 技术债 | techdebt | 0-100 | 代码混乱度，越高bug越多 |
| 薪资 | salary | 0-∞ | 月收入，用于恢复资源 |
| 同事关系 | colleaguerel | 0-100 | 人际关系，影响协作效率 |

#### 2.2 游戏循环

```
开始 → 分配行动点 → 事件触发 → 资源变动 → 检查状态 → 新循环
        ↑                                              |
        └──────────────── 30天后评估 ──────────────────┘
```

#### 2.3 职业路径

- **实习生**: 低薪低压力，学习期
- **初级开发**: 标准起点
- **中级开发**: 技术债管理关键期
- **高级开发**: 高薪高压
- **架构师**: 技术债清零责任
- **技术负责人**: 全面管理，最容易崩溃

### 3. 核心玩法

#### 3.1 每日行动 (5个行动点)

| 行动 | 消耗 | 效果 |
|------|------|------|
| 写代码 | 2AP | +进度, +techdebt |
| 重构 | 3AP | -techdebt, -sanity |
| 开会 | 1AP | 随机: ±colleaguerel |
| 摸鱼 | 1AP | +sanity, -进度 |
| 社交 | 2AP | ++colleaguerel |
| 学习 | 2AP | 解锁新技能 |
| 加班 | +2AP | -sanity, +进度 |

#### 3.2 随机事件系统

**日常事件**:
- 线上故障 → -sanity, +techdebt
- 需求变更 → --sanity
- 同事离职 → -colleaguerel
- 加薪 → +salary
- 团建 → ++colleaguerel, +/-sanity
- 代码 review → -techdebt, 可能-colleaguerel

**Boss 事件**:
- 突然的需求评审
- 周末紧急上线
- 年终 360 评估

#### 3.3 离职判定

**强制离职**:
- sanity = 0 (精神崩溃)
- techdebt = 100 (系统崩溃)
- colleaguerel = 0 (被孤立解雇)

**主动离职**:
- 玩家随时可选择离职
- 触发结局计算

**结局类型**:
- 财务自由 (salary 储蓄 > 目标)
-  burnout 休养 (sanity 恢复)
- 转行成功 (新职业)
- 创业 (高风险高回报)
- 彻底崩溃 (bad ending)

### 4. 技术需求

#### 4.1 技术栈

- **引擎**: 优先考虑 Godot / Unity WebGL / 纯 Web (Canvas)
- **语言**: GDScript / C# / TypeScript
- **存储**: LocalStorage (进度) / 可选后端

#### 4.2 架构要求

- 事件系统: 可配置 JSON
- 存档系统: 可导出/导入
- 成就系统: 多周目解锁
